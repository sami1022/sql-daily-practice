## Day 81 – Transactions & Locks Recap (Conceptual Explanation)

Transactions and locks ensure
safe and reliable database operations.

They are essential
for multi-user environments.

---

### Transactions Recap

A transaction is a group of operations
executed as a single unit.

Key commands:

- BEGIN → start transaction
- COMMIT → save changes
- ROLLBACK → undo changes

---

### ACID Properties Recap

ACID ensures:

- Atomicity → all or nothing
- Consistency → valid data
- Isolation → no interference
- Durability → permanent storage

---

### Locks Recap

Locks control access
to database rows.

Types:

- Shared Lock → read only
- Exclusive Lock → write control

---

### Concurrency Control

Concurrency control ensures:

- Multiple users can work safely
- Data remains consistent
- Conflicts are avoided

---

### Why These Concepts Matter

They are critical in:

- Banking systems
- Financial applications
- High-traffic platforms
- Multi-user systems

---

### Interview Importance

Very common questions:

- What is a transaction?
- What are ACID properties?
- Difference between COMMIT and ROLLBACK
- What are locks?

---

### Summary

Transactions + Locks provide:

- Data safety
- Error handling
- Consistency
- Reliability

These are core database concepts.
