## Day 91 – Interview SQL Practice (Conceptual Explanation)

This day focuses on solving
real interview-style SQL problems.

---

### Key Concepts Covered

1. JOIN + NULL filtering
2. Subqueries
3. Aggregation (GROUP BY)
4. HAVING clause
5. Sorting and limiting results

---

### Problem Types

- Anti-joins (customers without orders)
- Ranking problems (second highest salary)
- Duplicate detection
- Top-N queries
- User activity metrics

---

### Why This Is Important

These are:

- Frequently asked interview questions
- Used in real-world analytics
- Core for Data Analyst roles

---

### Interview Tips

- Always handle NULL values carefully
- Prefer EXISTS over NOT IN when needed
- Use GROUP BY + HAVING for duplicates
- Practice optimization thinking

---

### Summary

This practice strengthens:

- Problem-solving skills
- SQL logic building
- Interview readiness

This is a critical step before final evaluation.
