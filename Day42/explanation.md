SQL functions are used to manipulate,
format, and analyze data values.

They are essential for data cleaning,
analysis, and business reporting.

---

### String Functions
- UPPER and LOWER standardize text
- CONCAT joins multiple strings
- LENGTH measures string size
- Useful for formatting and validation

---

### Pattern Matching (LIKE)
- % matches multiple characters
- _ matches a single character
- Used for flexible text searching
- Common in search and filtering

---

### Numeric Functions
- ROUND controls decimal precision
- CEIL rounds numbers upward
- FLOOR rounds numbers downward
- Used in financial calculations

---

### Date Functions
- DATE extracts date from timestamp
- EXTRACT retrieves specific date parts
- Enables time-based filtering

---

### Date Arithmetic
- INTERVAL adds or subtracts time
- Calculates durations
- Used in SLA and trend analysis

---

### Best Practices
- Standardize data early
- Avoid heavy functions in WHERE on large tables
- Be mindful of performance
- Use correct data types

---

### Common Mistakes
- Ignoring case sensitivity
- Rounding too early
- Misusing LIKE wildcards
- Confusing DATE and TIMESTAMP

---

### Real-World Applications
- Analytics dashboards
- Data cleaning pipelines
- Business reporting
- Interview SQL problems

---

### Summary
- Functions transform and analyze data
- Core skill for SQL analysts
- Used in almost every real query
- Must-know for interviews
